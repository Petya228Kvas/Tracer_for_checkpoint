# T&C - Trace & Checkpoint

##  Описание

**T&C** (Trace & Checkpoint) — это трассировщик процессов, который:
- Подключается к целевому процессу
- Создаёт дочерний процесс с помощью системного вызова `fork()`
- Вызывает утилиту [CRIU](https://criu.org/) для создания контрольной точки (checkpoint)

##  Требования

- **Компилятор**: `gcc`
- **Утилита**: `criu` (установлена в системе)
- **Архитектура**: `x86_64`
- **ОС**: Linux

##  Установка

Клонируйте репозиторий и выполните сборку:

```bash
git clone https://github.com/Petya228Kvas/Tracer_for_checkpoint.git
cd Tracer_for_checkpoint
make
```

##  Использование

Запустите трассировщик, указав PID целевого процесса:

```bash
./tracer [PID]
```

где `[PID]` — идентификатор целевого процесса.

###  Важное замечание

Чтобы при создании контрольной точки не оставалось **зомби-дочерних процессов**, в начале функции `main()` целевой программы необходимо добавить:

```c
#include <signal.h>

int main(int argc, char *argv[]) {
    signal(SIGCHLD, SIG_IGN);  // Игнорируем сигналы завершения дочерних процессов
    // ... ваш код
}
```

###  Пример

В репозитории предоставлена демонстрационная программа `test.c`, которую можно использовать для тестирования работы трассировщика.

Пример запуска:

```bash
# Запускаем тестовую программу
./test &
# Получаем её PID
ps aux | grep test
# Запускаем трассировщик
./tracer [PID_процесса]
```

##  Восстановление из контрольной точки

После создания контрольной точки (сохраняется в директории `dump/`), для восстановления выполнения программы используйте:

```bash
sudo criu restore --shell-job -d -D dump/
```

### Параметры:
- `--shell-job` — поддержка shell-задач
- `-d` — демонизация процесса
- `-D dump/` — директория с контрольной точкой
